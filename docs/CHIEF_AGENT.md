# Chief Agent - الوكيل الرئيسي

## نظرة عامة

Chief Agent هو وكيل ذكي يعمل كنائب للمالك في Synaptic Space. يقوم بمراقبة النظام تلقائياً واتخاذ قرارات بناءً على السياسات المحددة.

## القدرات الأساسية

### 1. إدارة المحتوى
- حذف الرسائل المخالفة
- إخفاء المحتوى المشبوه
- حذف الخلايا (المجموعات) المخالفة
- تجميد الخلايا مؤقتاً

### 2. إدارة المستخدمين
- حظر المستخدمين المخالفين
- إرسال تحذيرات
- تتبع السلوك المشبوه

### 3. تحليل الأخطاء (جديد)
- قراءة Support Tickets من قاعدة البيانات
- البحث في GitHub عن الكود المشتبه به
- تحليل الأخطاء باستخدام AI متقدم (Claude Sonnet 4.5)
- اقتراح حلول بناءً على الكود الفعلي
- اكتشاف الأنماط المتكررة

## كيفية عمل تحليل الأخطاء

### 1. جمع البيانات
عند الضغط على "تحليل التذاكر الحالية":
- يجلب آخر 10 تذاكر مفتوحة من `support_tickets`
- يستخرج الكلمات المفتاحية من رسالة الخطأ
- يبحث في GitHub عن الملفات المحتملة

### 2. البحث في GitHub
\`\`\`typescript
// يستخدم GitHub API للبحث
const results = await octokit.rest.search.code({
  q: `${keyword} repo:${owner}/${repo}`,
})
\`\`\`

### 3. التحليل بالذكاء الاصطناعي
- يرسل الخطأ + الكود المشتبه به إلى Claude
- يحصل على تحليل شامل:
  - ملخص المشكلة
  - الأسباب المحتملة (3-5 أسباب)
  - مستوى الخطورة
  - الحلول المقترحة (3-5 حلول)
  - مستوى الثقة (0-100%)

### 4. عرض النتائج
- الأخطاء الحرجة في الأعلى
- الأنماط المتكررة
- التوصيات العامة

## الإعدادات المطلوبة

### GitHub Token (اختياري)
للوصول إلى مستودعات خاصة:

1. اذهب إلى GitHub Settings > Developer settings > Personal access tokens
2. أنشئ token جديد مع صلاحية `repo` (للمستودعات الخاصة) أو بدون صلاحيات (للعامة)
3. أضف المتغيرات:
   \`\`\`env
   GITHUB_TOKEN=ghp_xxxxxxxxxxxxx
   GITHUB_OWNER=your-username
   GITHUB_REPO=microcosm
   \`\`\`

### AI Model
يستخدم النظام حالياً:
- `anthropic/claude-sonnet-4.5` للتحليل المتقدم
- `xai/grok-beta` للمحادثة العامة

## أمثلة على الاستخدام

### مثال 1: تحليل خطأ في الإشعارات
\`\`\`
الخطأ: "Push notifications not working"
النتيجة:
- الملفات المشتبه بها: lib/firebase-admin-server.ts, app/api/notifications/send/route.ts
- السبب المحتمل: حلقة تسلسلية في إرسال الإشعارات
- الحل المقترح: استخدام Promise.all للإرسال المتوازي
- الثقة: 92%
\`\`\`

### مثال 2: اكتشاف نمط متكرر
\`\`\`
النمط: lib/notifications-server.ts
التكرار: 5 مرات
التوصية: "يوجد أخطاء متكررة في هذا الملف - يُنصح بمراجعته"
\`\`\`

## نصائح للمطورين

### استخدام التحليل بفعالية
1. **قبل إصلاح خطأ**: شغّل التحليل لفهم السبب الجذري
2. **بعد deployment**: راقب التذاكر الجديدة وحللها فوراً
3. **للأخطاء المعقدة**: استخدم التحليل للحصول على اقتراحات متعددة

### تحسين دقة التحليل
- اكتب تذاكر واضحة مع stack traces
- أضف خطوات لإعادة إنتاج الخطأ
- حدد المتصفح/الجهاز إن أمكن
